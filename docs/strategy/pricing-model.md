## 1. 課金モデル比較

| モデル | 説明 | メリット | デメリット | 適するプロダクト |
|--------|------|---------|-----------|----------------|
| **Flat Rate（定額）** | 全機能を固定価格で提供 | シンプル、予測可能 | 大口顧客を取りこぼす | 機能が少ないプロダクト |
| **Per Seat（ユーザー課金）** | ユーザー数に応じた課金 | スケールしやすい | シート数を絞られる | コラボレーションツール |
| **Usage-based（従量課金）** | 使用量に応じた課金 | 公平、小規模でも始めやすい | 収益予測が困難 | API、ストレージ、通信 |
| **Hybrid（ハイブリッド）** | 基本料金 + 従量課金 | 安定収益 + 成長余地 | 価格体系が複雑 | 多機能SaaS |

## 3. 年間割引戦略

### 3.1 割引設計

| プラン | 月額 | 年額（月あたり） | 年間割引 | お得額 |
|--------|------|---------------|---------|--------|
| Pro | ¥___/月 | ¥___/月 | **2ヶ月分無料（約17%OFF）** | ¥___/年 |
| Enterprise | ¥___/月 | ¥___/月 | **2ヶ月分無料（約17%OFF）** | ¥___/年 |

### 3.2 計算式

```
年額 = 月額 x 10ヶ月分（2ヶ月分無料）
割引率 = 2/12 = 約16.7%（≒17% OFF と表示）

例: 月額 ¥2,980 の場合
  月払い: ¥2,980 x 12 = ¥35,760/年
  年払い: ¥2,980 x 10 = ¥29,800/年
  お得額: ¥5,960/年（2ヶ月分無料）
```

## 4. 価格表示の心理学

### 4.1 アンカリング効果

最初に高い価格を見せることで、他のプランが割安に感じられる効果。

```
表示順序（推奨）:
Enterprise → Pro（推奨） → Free

理由: 高価格を先に見せることで、Pro プランの「お得感」を強調
```

### 4.2 価格表示のベストプラクティス

| テクニック | 説明 | 例 |
|-----------|------|-----|
| **端数価格** | 心理的に安く感じる | ¥2,980（¥3,000ではなく） |
| **月額換算表示** | 年額を月あたりで表示 | 「月あたり¥2,483」 |
| **節約額の強調** | 年払い時の節約額を表示 | 「年間¥5,960 おトク」 |
| **無料プランとの対比** | 有料プランの追加価値を明示 | 「Freeに加えて+20の機能」 |
| **推奨バッジ** | 最も選んでほしいプランを強調 | 「最も人気」「おすすめ」 |

## 5. Van Westendorp 価格感度調査

### 5.1 4つの質問

ユーザーに以下の4つの質問をし、適正価格帯を特定する。

| # | 質問 | 説明 |
|---|------|------|
| Q1 | 「いくらから**高すぎる**と感じますか？」 | 購入しないほど高い価格 |
| Q2 | 「いくらから**高い**と感じますか？」 | 高いが購入検討する価格 |
| Q3 | 「いくらから**安い**と感じますか？」 | お得に感じる価格 |
| Q4 | 「いくらから**安すぎる**と感じますか？」 | 品質に不安を感じる価格 |

### 5.2 調査設計

| 項目 | 内容 |
|------|------|
| **対象** | ICP に合致するユーザー |
| **サンプル数** | 最低 100名（理想は 200名+） |
| **調査方法** | オンラインアンケート（Typeform / Google Forms） |
| **前提条件** | プロダクトの機能一覧を提示した上で回答 |
