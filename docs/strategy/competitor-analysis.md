## 3. 間接競合（Indirect Competitors）

間接競合とは、同じ課題を異なる方法で解決しているプロダクトや手段。

| 間接競合 | 解決方法 | メリット | デメリット | 自社の優位性 |
|---------|---------|---------|-----------|-------------|
| Excel / スプレッドシート | 手動管理 | 無料、柔軟 | スケールしない、属人化 | |
| 紙 / ホワイトボード | アナログ管理 | 導入コストゼロ | 共有困難、検索不可 | |
| 社内開発ツール | カスタム開発 | 要件に完全一致 | 開発・保守コスト大 | |
| 複数SaaSの組み合わせ | ツール併用 | 各領域に特化 | 統合コスト、学習コスト | |
| コンサルティング | 外部委託 | 専門知識 | 高コスト、依存 | |
