# エグゼクティブダッシュボード設計（Phase 74）

## 0. 概要

ファウンダーが週次・月次で確認すべき経営指標を一元管理するダッシュボード設計。
North Star Metric を中心に、収益・成長・効率性の指標を階層的に可視化する。

---

## 1. North Star Metric

### 1.1 定義

| 項目 | 内容 |
|------|------|
| 指標名 | Weekly Active Teams（WAT） |
| 定義 | 過去7日間にアクティビティのあった workspace_id の数 |
| 計測方法 | `SELECT COUNT(DISTINCT workspace_id) FROM activity_logs WHERE created_at > NOW() - INTERVAL '7 days'` |
| 更新頻度 | 日次（毎朝 06:00 JST） |

### 1.2 目標設定

| 期間 | 目標 | 根拠 |
|------|------|------|
| Q1 2026 | 50 WAT | 初期ユーザー獲得フェーズ |
| Q2 2026 | 150 WAT | PMF検証完了目標 |
| Q3 2026 | 500 WAT | グロースフェーズ突入 |
| Q4 2026 | 1,500 WAT | スケーリングフェーズ |

### 1.3 WAT 向上のレバー

```
WAT = 新規チーム登録数 + 既存チーム継続率 - チャーンチーム数

向上施策:
  - 新規チーム登録: マーケティング + リファラル + セルフサーブ
  - 既存チーム継続: オンボーディング改善 + 価値体験の早期化
  - チャーン防止: ヘルススコア監視 + 早期介入
```

## 2. 経営指標体系

### 2.1 収益指標（Revenue）

| 指標 | 計算式 | 目標（Q2 2026） |
|------|--------|-----------------|
| MRR（月次経常収益） | SUM(月額サブスクリプション) | 300万円 |
| ARR（年次経常収益） | MRR x 12 | 3,600万円 |
| ARPU（ユーザー単価） | MRR / アクティブチーム数 | 20,000円 |
| Net Revenue Retention | (期初MRR + 拡張 - 縮小 - チャーン) / 期初MRR | 110% |

### 2.2 成長指標（Growth）

| 指標 | 計算式 | 目標（月次） |
|------|--------|-------------|
| 新規顧客数 | 当月新規サブスクリプション開始数 | 30チーム |
| チャーン率 | 解約チーム数 / 期初チーム数 | < 5% |
| MRR成長率 | (当月MRR - 前月MRR) / 前月MRR | > 15% |
| Trial-to-Paid転換率 | 有料化チーム / トライアル開始チーム | > 25% |

### 2.3 効率性指標（Efficiency）

| 指標 | 計算式 | 目標 |
|------|--------|------|
| LTV（顧客生涯価値） | ARPU / 月次チャーン率 | 400,000円 |
| CAC（顧客獲得単価） | マーケ費用 / 新規顧客数 | < 100,000円 |
| LTV/CAC比率 | LTV / CAC | > 3.0 |
| CAC回収期間 | CAC / ARPU | < 6ヶ月 |

### 2.4 財務指標（Financial）

| 指標 | 計算式 | 目標 |
|------|--------|------|
| バーンレート | 月次支出総額 | < 500万円 |
| ランウェイ | 残高 / バーンレート | > 18ヶ月 |
| グロスマージン | (収益 - 原価) / 収益 | > 70% |
| Rule of 40 | 成長率 + 利益率 | > 40% |

## 3. 指標階層図

```
                    ┌─────────────────────┐
                    │   North Star Metric  │
                    │ Weekly Active Teams  │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
     ┌────────▼────────┐ ┌────▼─────┐ ┌────────▼────────┐
     │  先行指標        │ │ 同時指標  │ │  遅行指標        │
     │ (Leading)       │ │(Current) │ │ (Lagging)       │
     └────────┬────────┘ └────┬─────┘ └────────┬────────┘
              │                │                │
    ┌─────────┼──────┐    ┌───┼───┐    ┌───────┼────────┐
    │         │      │    │   │   │    │       │        │
  新規登録  Trial  NPS  DAU WAT MAU  MRR   チャーン  LTV
  数       転換率      /WAU         成長率   率
```

### 先行指標（Leading Indicators）

将来のパフォーマンスを予測する指標:
- **新規登録数**: パイプラインの健全性
- **Trial-to-Paid転換率**: プロダクト価値の実証
- **NPS**: 口コミ成長の可能性

### 同時指標（Current Indicators）

現在の状態を示す指標:
- **DAU/WAU比率**: エンゲージメント深度
- **WAT**: North Star Metric
- **MAU**: 月次アクティブユーザー

### 遅行指標（Lagging Indicators）

過去の結果を示す指標:
- **MRR成長率**: 収益の成長トレンド
- **チャーン率**: 顧客維持の結果
- **LTV**: 長期的な顧客価値

### 5.2 差異分析テンプレート

```
差異分析レポート（月次）
========================

対象月: YYYY年MM月

   - 予算: XXXX万円

   | 項目 | 予算 | 実績 | 差異 | 原因 |
   |------|------|------|------|------|
   |      |      |      |      |      |

   - [差異が10%以上の項目について是正措置を記載]

   - [翌月の予算達成見込みを記載]
```

### 5.3 予測更新ルール

| トリガー | アクション |
|---------|-----------|
| 実績が予算の90-110% | 予測変更なし |
| 実績が予算の110-120% | 月次レビューで要因分析、翌月調整 |
| 実績が予算の120%超 | 即座に経費レビュー、予算再編成 |
| 実績が予算の80%未満 | 未執行理由の確認、リソース再配分検討 |
| MRR目標未達が2ヶ月連続 | グロース戦略の見直し |

### 7.2 キャッシュアラート

| 条件 | レベル | アクション |
|------|--------|-----------|
| ランウェイ > 18ヶ月 | 正常 | 通常運営 |
| ランウェイ 12-18ヶ月 | 注意 | 資金調達の準備を開始 |
| ランウェイ 6-12ヶ月 | 警告 | 資金調達を本格始動、支出の見直し |
| ランウェイ < 6ヶ月 | 緊急 | 緊急コスト削減、ブリッジファイナンス検討 |
| バーンレート > 予算120% | 警告 | 経費レビュー、未承認支出の停止 |

### 7.3 緊急時キャッシュ保全策

```
ランウェイ < 6ヶ月の場合:

1. 即時対応（1週間以内）
   - 不要な SaaS サブスクリプションの解約
   - マーケティング予算の凍結
   - 新規採用の凍結

2. 短期対応（1ヶ月以内）
   - 外注契約の見直し・縮小
   - インフラコストの最適化
   - 年間契約の月次契約への切り替え検討

3. 中期対応（3ヶ月以内）
   - ブリッジファイナンスの実行
   - ピボットの検討
   - M&A/事業売却の検討
```
