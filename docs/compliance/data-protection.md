# データ保護設計書（Phase 78）

> FDC Modular Starter における GDPR、個人情報保護法、CCPA 等のデータ保護規制への対応設計を定義する。

---

## 1. 規制概要

### 1.1 対象規制一覧

| 規制 | 対象 | 最大制裁金 | 優先度 |
|------|------|-----------|--------|
| 個人情報保護法（日本） | 国内ユーザーのデータ | 1億円 | 高 |
| GDPR（EU一般データ保護規則） | EU居住者のデータ | 2,000万EUR または全世界売上4% | 中 |
| CCPA（カリフォルニア州消費者プライバシー法） | カリフォルニア州居住者のデータ | 違反1件あたり$7,500 | 低 |

### 1.2 コンプライアンスロードマップ

| フェーズ | 対象規制 | 優先度 | 時期 | 概要 |
|---------|---------|--------|------|------|
| Phase 1 | 日本個人情報保護法 | 高 | Q1 2026 | 国内法準拠（必須） |
| Phase 2 | GDPR 基本対応 | 中 | Q2 2026 | EU展開準備 |
| Phase 3 | GDPR 完全準拠 + CCPA | 低 | Q3-Q4 2026 | グローバル展開 |

## 2. GDPR 対応

### 2.1 データ保護6原則

| # | 原則 | 説明 | FDC での対応 |
|---|------|------|-------------|
| 1 | 適法性・公正性・透明性 | 合法な根拠に基づき、透明性をもってデータを処理 | 同意管理システム + プライバシーポリシー |
| 2 | 目的制限 | 特定された明確な目的のためにのみデータを収集 | 利用目的の明示（プライバシーポリシー第2条） |
| 3 | データ最小化 | 目的に必要なデータのみを収集 | Google OAuth で最小限のスコープのみ要求 |
| 4 | 正確性 | データを正確かつ最新の状態に維持 | プロフィール編集機能 + OAuth 同期 |
| 5 | 保存制限 | 必要な期間のみデータを保持 | データ保持ポリシー + 自動削除バッチ |
| 6 | 完全性・機密性 | 適切なセキュリティ措置でデータを保護 | 暗号化 + RLS + アクセス制御 |

### 2.2 処理の法的根拠

| 法的根拠 | 適用場面 | 具体例 |
|---------|---------|--------|
| 同意（Consent） | マーケティング、分析 Cookie | ニュースレター配信、PostHog トラッキング |
| 契約履行（Contract） | サービス提供に必要な処理 | アカウント管理、タスクデータ保存 |
| 法的義務（Legal Obligation） | 法令で義務付けられた処理 | 税務記録の保持、取引履歴の保管 |
| 正当な利益（Legitimate Interest） | セキュリティ、不正防止 | ログ記録、異常検知、サービス改善 |

## 3. データ主体の権利

### 3.1 権利一覧

| # | 権利 | 説明 | 対応期限 | 実装方法 |
|---|------|------|---------|---------|
| 1 | アクセス権 | 自身のデータの閲覧・コピー取得 | 30日以内 | データエクスポート機能 |
| 2 | 訂正権 | 不正確なデータの修正を要求 | 30日以内 | プロフィール編集画面 |
| 3 | 消去権（忘れられる権利） | データの削除を要求 | 30日以内 | アカウント削除機能 |
| 4 | 処理制限権 | データ処理の制限を要求 | 30日以内 | アカウント一時停止機能 |
| 5 | ポータビリティ権 | 機械可読形式でのデータ受領 | 30日以内 | JSON エクスポート |
| 6 | 異議申立権 | 特定の処理への異議 | 30日以内 | 同意撤回 + 問い合わせ |

### 3.5 削除対象データマトリクス

| データ種別 | 処理 | 保持期間 | 根拠 |
|-----------|------|---------|------|
| プロフィール情報 | 削除 | 猶予期間後に即時削除 | 不要 |
| タスク・アクティビティデータ | 削除 | 猶予期間後に即時削除 | 不要 |
| アクティビティログ | 匿名化 | user_id を null に置換 | セキュリティ監査 |
| 決済履歴 | 保持 | 7年間 | 税法上の義務（電子帳簿保存法） |
| サポート履歴 | 匿名化 | ユーザー情報を匿名化して保持 | サービス改善 |
| 監査ログ | 匿名化 | user_id をハッシュに置換 | コンプライアンス |
| 同意記録 | 保持 | 5年間 | GDPR 立証義務 |

### 3.6 削除処理の実装

```sql
-- アカウント削除バッチ（30日猶予期間後に実行）
-- 1. プロフィール削除
DELETE FROM profiles WHERE user_id = :user_id;

-- 2. アクティビティデータ削除
DELETE FROM tasks WHERE user_id = :user_id;
DELETE FROM leads WHERE workspace_id IN (
  SELECT workspace_id FROM workspace_members
  WHERE user_id = :user_id AND role = 'OWNER'
);

-- 3. アクティビティログの匿名化
UPDATE activity_logs
SET user_id = NULL,
    ip_address = NULL,
    user_agent = NULL,
    metadata = metadata - 'email' - 'name'
WHERE user_id = :user_id;

-- 4. 監査ログの匿名化
UPDATE audit_logs
SET user_id = NULL,
    ip_address = NULL,
    user_agent = NULL
WHERE user_id = :user_id;

-- 5. 同意記録は保持（GDPR 立証義務）
-- user_consents は削除しない

-- 6. 決済履歴は保持（税法上の義務）
-- billing_history は削除しない（7年保持）
```

## 4. 日本個人情報保護法対応

### 4.1 必要要件

| 要件 | 説明 | FDC での対応 |
|------|------|-------------|
| 利用目的の特定 | 個人情報の利用目的を具体的に特定 | プライバシーポリシーに明記 |
| 取得時の通知 | 個人情報取得時に利用目的を通知 | 登録フロー + 同意チェックボックス |
| 安全管理措置 | 個人データの漏洩防止措置 | 暗号化 + RLS + アクセスログ |
| 第三者提供の制限 | 本人の同意なく第三者に提供しない | 同意管理 + 委託先管理 |
| 開示請求対応 | 本人からの開示請求に対応 | データエクスポート機能 |

### 4.2 安全管理措置（4カテゴリ）

| カテゴリ | 措置内容 | 具体的な対策 |
|---------|---------|-------------|
| 組織的安全管理措置 | 体制整備、規程策定 | 情報セキュリティポリシー策定、責任者任命、インシデント対応手順 |
| 人的安全管理措置 | 従業員の教育、監督 | セキュリティ研修、秘密保持契約、アクセス権限の最小化 |
| 物理的安全管理措置 | 機器、記録媒体の管理 | クラウドサービス利用（物理サーバー管理はプロバイダーに委託） |
| 技術的安全管理措置 | アクセス制御、暗号化 | RLS、CSP、TLS 暗号化、入力サニタイズ、監査ログ |

### 5.1 同意種別と撤回オプション

| 同意種別 | 撤回方法 | 撤回後の影響 |
|---------|---------|-------------|
| 利用規約 | アカウント削除のみ | サービス利用不可 |

| Cookie（分析） | Cookie設定画面 | 分析データ収集停止 |
| Cookie（マーケティング） | Cookie設定画面 | 広告トラッキング停止 |

## 6. 委託先（サブプロセッサー）管理

### 6.1 委託先一覧

| サービス | 事業者 | 委託データ | データ所在地 | DPA 締結 |
|---------|--------|-----------|-------------|---------|
| Supabase | Supabase Inc. | ユーザーデータ全般、認証情報 | AWS 東京リージョン | 済 |
| Vercel | Vercel Inc. | アクセスログ、デプロイ成果物 | グローバル（エッジ） | 済 |
| Stripe | Stripe Inc. | 決済情報、顧客メタデータ | 米国（PCI DSS 準拠） | 済 |
| Resend | Resend Inc. | メールアドレス、送信内容 | 米国 | 済 |

### 6.2 委託先管理ルール

| ルール | 内容 |
|-------|------|
| 事前通知 | 新規委託先追加時は30日前にユーザーに通知 |
| DPA 確認 | データ処理契約（DPA）の締結を必須とする |
| 年次レビュー | 年1回、全委託先のセキュリティ状況を確認 |
| 公開リスト | 委託先一覧をプライバシーポリシーで公開 |
| 変更対応 | 委託先変更に異議がある場合はアカウント削除で対応 |

### 6.3 DPA チェックリスト

```
委託先評価チェック項目:

[ ] SOC 2 Type II または ISO 27001 認証の確認
[ ] データ処理契約（DPA）の締結
[ ] データ所在地の確認（適切な国/リージョン）
[ ] 暗号化の実装状況（転送時・保存時）
[ ] インシデント通知プロセスの確認
[ ] データ削除・返却ポリシーの確認
[ ] 再委託の有無と管理体制の確認
```

## 7. データ処理記録

### 7.1 必要記録項目

| 項目 | 説明 | 具体例 |
|------|------|--------|
| 処理目的 | データ処理の目的 | サービス提供、分析、マーケティング |
| データカテゴリ | 処理するデータの種類 | アカウント情報、利用データ、決済データ |
| 受領者 | データの受領者/共有先 | Supabase、Stripe、Vercel、Resend |
| 保持期間 | データの保持期間 | アクティブ期間中 + 削除後30日 |
| セキュリティ措置 | 適用されるセキュリティ措置 | 暗号化、RLS、アクセス制御 |

### 7.2 処理記録テーブル

```sql
CREATE TABLE data_processing_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  processing_purpose TEXT NOT NULL,         -- 処理目的
  data_categories TEXT[] NOT NULL,           -- データカテゴリ
  data_subjects TEXT NOT NULL,               -- データ主体の種類
  recipients TEXT[],                         -- 受領者
  third_country_transfers TEXT[],            -- 第三国移転先
  retention_period TEXT NOT NULL,            -- 保持期間
  security_measures TEXT[] NOT NULL,         -- セキュリティ措置
  legal_basis TEXT NOT NULL,                 -- 法的根拠
  dpia_required BOOLEAN DEFAULT false,       -- DPIA 必要性
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 7.3 処理活動の記録例

| 処理活動 | 目的 | カテゴリ | 法的根拠 | 保持期間 |
|---------|------|---------|---------|---------|
| アカウント管理 | サービス提供 | アカウント情報 | 契約履行 | アカウント存続中 |
| タスク管理 | サービス提供 | 利用データ | 契約履行 | アカウント存続中 |
| 決済処理 | 料金徴収 | 決済データ | 契約履行 | 7年間（税法） |
| アクセス分析 | サービス改善 | 利用データ | 正当な利益/同意 | 2年間 |
| メール通知 | マーケティング | メールアドレス | 同意 | 同意撤回まで |
| セキュリティ監視 | 不正防止 | アクセスログ | 正当な利益 | 1年間 |
